# Descripción general

El Service Domain es la unidad básica de funcionalidad en BIAN.

# Propósito de negocio

Representar una responsabilidad única y autónoma.

# Capacidades principales

- Servicio independiente
- Función específica
- Interacción mediante APIs

# Relación con otros dominios

Interactúa con otros Service Domains formando el sistema bancario.

# Aplicabilidad en arquitectura

- Microservicios
- APIs bancarias
- Modularización

# Observaciones

- Son atómicos
- No se solapan
- Cubren todo el banco :contentReference[oaicite:6]{index=6}
